Showcase

MentatEngine is not only a rendering sandbox. It also includes playable and editable scenes that demonstrate how the ECS, rendering pipeline, editor workflow and scripting system fit together.

PongGame Scene

One of the most representative showcase scenes in the project is PongGame. It works well as a demonstration because it combines several important engine systems in one place.

  • ECS entities and components
  • scene loading through the editor flow
  • transform-based gameplay objects
  • runtime testing with the Player panel
  • Lua or script-driven logic attached to entities
PongGame is a strong example to highlight on the website because it shows that the engine can be used not only to render objects, but also to prototype gameplay scenes and test logic inside the editor.

What the Game Demonstrates

Even as a small game, Pong is a very good engine showcase because it tends to involve:

  • moving entities such as paddles and ball
  • component-based scene structure
  • collision or gameplay response logic
  • runtime state changes
  • script-controlled behavior

In a documentation website, this kind of scene is valuable because it turns abstract systems like Entity, TransformComponent or ScriptComponent into something visible and easy to explain.

Where It Fits in the Engine

The showcase scene connects naturally with the rest of the engine website:

  • Getting Started explains how to build and launch the engine
  • How To's explains the ECS workflows used to create entities and components
  • Editor Panels explains how to load, inspect and test the scene from the editor
  • Documentation gives the API-level view of the classes involved

Suggested Workflow to Try the Game

  • Open the engine editor
  • Go to the title scene or load flow
  • Load PongGame
  • Inspect the entities in the hierarchy
  • Check the scripts attached to gameplay entities
  • Press Play in the Player panel
  • Observe runtime behavior, then use Stop to restore the scene state
This scene is especially useful for showing the difference between authoring mode and play mode inside the editor.

Why It Matters for the Website

The presence of a small playable scene makes the site feel more like the documentation of a real engine and less like a static API dump. It gives visitors a concrete example of what MentatEngine can already do.
