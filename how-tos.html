How To's

Basic recipes for working with MentatEngine. These examples are based on the engine examples and the current ECS workflow using entities and components.

1. Create a Core

Start by creating the engine core, initializing the renderer and selecting the render type.

std::optional<ME::Core> maybe_core = ME::Core::create(
    1280, 720, "My Scene", ME::APIType::OpenGL
);

if (!maybe_core) return -1;

ME::Core core = std::move(maybe_core.value());
core.InitRenderer(true);
core.GetGraphicAPI()->StartRenderers();
core.GetGraphicAPI()->SetRenderType(ME::RenderType::Phong);
Use this as the usual entry point before creating entities or rendering meshes.

2. Create an Entity

An entity is the base object in the ECS. Creating one gives it an ID automatically.

ME::Entity entity;
unsigned long id = entity.GetID();
By default, entities get a TagComponent automatically with a name like "Entity X".

3. Add a TransformComponent

Transforms store position, rotation and scale.

entity.AddComponent<ME::TransformComponent>();

ME::TransformComponent* tc = entity.GetComponent<ME::TransformComponent>();
tc->SetPosition({0.0f, 1.0f, 0.0f});
tc->SetRotation({0.0f, 0.0f, 0.0f});
tc->SetScale({1.0f, 1.0f, 1.0f});
This is the component you will usually combine with render, light and hierarchy logic.

4. Add a RenderizableComponent

To render geometry, add a RenderizableComponent with vertices, indices, color and optionally a mesh path.

ME::LoadedDrawable mesh =
    ME::MeshLoader::LoadObjPositionsFlat("../../deps/resources/horse.obj");

ME::Entity model;
model.AddComponent<ME::RenderizableComponent>(
    mesh.vertices.data(),
    mesh.vertices.size(),
    mesh.indexes.data(),
    mesh.indexes.size(),
    Vec3{1.0f, 0.0f, 0.0f},
    "../../deps/resources/horse.obj"
);

model.AddComponent<ME::TransformComponent>();
This is the clearest render workflow in the examples I reviewed.

5. Move an Entity with Input

Bind actions to keys, read them every frame, and update the TransformComponent.

ME::Input input(core.window());
input.BindEditorKey(ME::Input::MoveUp, ME::Input::W);
input.BindEditorKey(ME::Input::MoveDown, ME::Input::S);
input.BindEditorKey(ME::Input::MoveLeft, ME::Input::A);
input.BindEditorKey(ME::Input::MoveRight, ME::Input::D);

ME::TransformComponent* tc =
    ME::ECS::GetComponent<ME::TransformComponent>(entity.GetID());

Vec3 pos = tc->Position();
if (input.checkActionPressed(ME::Input::MoveUp))    pos.y_ += core.DeltaTime();
if (input.checkActionPressed(ME::Input::MoveDown))  pos.y_ -= core.DeltaTime();
if (input.checkActionPressed(ME::Input::MoveRight)) pos.x_ += core.DeltaTime();
if (input.checkActionPressed(ME::Input::MoveLeft))  pos.x_ -= core.DeltaTime();

tc->SetPosition(pos);
This pattern is used repeatedly in the examples.

6. Add a Fly Camera

The CameraComponent supports fly movement through UpdateFly.

ME::Entity cameraEntity;
cameraEntity.AddComponent<ME::CameraComponent>();

ME::Input input(core.window());
input.BindEditorKey(ME::Input::Action::FlyEnable,   ME::Input::InputKey::MB_Right);
input.BindEditorKey(ME::Input::Action::FlyForward,  ME::Input::InputKey::W);
input.BindEditorKey(ME::Input::Action::FlyBackward, ME::Input::InputKey::S);
input.BindEditorKey(ME::Input::Action::FlyLeft,     ME::Input::InputKey::A);
input.BindEditorKey(ME::Input::Action::FlyRight,    ME::Input::InputKey::D);
input.BindEditorKey(ME::Input::Action::FlyDown,     ME::Input::InputKey::E);
input.BindEditorKey(ME::Input::Action::FlyUp,       ME::Input::InputKey::Q);

auto* cam = cameraEntity.GetComponent<ME::CameraComponent>();
cam->UpdateFly(input, core.window(), core.DeltaTime());
In the shadow mapping example, the fly camera is updated every frame.

7. Add a LightComponent

Lights can be point, directional or spot, and can optionally cast shadows.

ME::Entity lightEntity;
auto& light = lightEntity.AddComponent<ME::LightComponent>();

light.SetType(ME::LightType::DIRECTIONAL);
light.SetEnabled(true);
light.SetDiffColor({1.0f, 1.0f, 1.0f});
light.SetSpecColor({1.0f, 1.0f, 1.0f});
light.SetDir({-0.2f, -1.0f, -0.3f});
light.SetCastsShadows(true);
For point lights and spotlights, you can also configure attenuation and cutoff values.

8. Attach a Lua Script

Scripts are handled with ScriptComponent and can expose engine types such as Entity, Transform, Input and ECS.

ME::Entity scripted;
auto& script = scripted.AddComponent<ME::ScriptComponent>(scripted.GetID());

script.setCodeByPath("../../deps/resources/scripts/myScript.lua");
script.EnableCode(true);
-- myScript.lua
function OnStart(dt, selfId)
    print("Script started", selfId)
end

function OnFrame(dt, selfId)
    local transform = ECS.GetTransformComp(selfId)
    if transform ~= nil then
        local pos = transform:Position()
        pos.y_ = pos.y_ + dt
        transform:SetPosition(pos)
    end
end
This section may need a small revision with you to document the exact recommended script lifecycle.

9. Set Parent-Child Relationships

Hierarchy is stored through the TransformComponent parent ID.

ME::Entity parent;
ME::Entity child;

parent.AddComponent<ME::TransformComponent>();
child.AddComponent<ME::TransformComponent>();

parent.SetParent(child);
The current implementation suggests the child entity stores the parent ID inside its TransformComponent.

10. Remove an Entity

Use the ECS directly to remove an entity and all its associated components.

ME::ECS::RemoveEntity(entity.GetID());
This is used directly in the ECS example when swapping visible entities.

11. Find an Entity by Tag

Entities get a TagComponent automatically and the ECS exposes lookup by tag.

unsigned long id = ME::ECS::GetEntityByTag("Player");
You can use tags to locate important scene entities without storing IDs everywhere.
