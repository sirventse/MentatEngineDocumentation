◆ tEnqueue()

template<class F, class... Args>
auto JobSystem::tEnqueue ( F && f,
Args &&... args ) -> std::future< std::invoke_result_t< F, Args... > >
inline

Enqueues a task for asynchronous execution in the thread pool.

This method wraps a function and its arguments into a packaged task, pushes it into the job queue, and notifies a worker thread.

Template Parameters
FThe type of the function or callable object.
ArgsThe types of the arguments to be passed to the function.
Parameters
fThe function to be executed by a worker thread.
argsThe specific arguments to forward to the function.
Returns
A std::future that will eventually hold the result of the function execution.
Exceptions
std::runtime_errorIf the job system has been stopped and cannot accept new tasks.
Note
This method is thread-safe as it uses an internal mutex to protect the task queue.

◆ worker()

void JobSystem::worker ( )

The main loop for worker threads.

This method runs on each thread in the pool. It waits for new tasks in the queue using a condition variable and executes them as they become available until the system is stopped.
