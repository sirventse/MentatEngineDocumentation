Editor Panels

A practical overview of the editor UI based on ImGuiManager.cpp. This page explains what each panel is for and shows a basic workflow for editing a scene using the ECS, Phong rendering and Shadow Map rendering.

Rendering Modes

The editor and examples work around two main render modes:

  • Phong: the standard lit rendering mode for general scene editing and material/light iteration.
  • Shadow Map: used when lights cast shadows and you want to validate lighting depth and shadow behavior.
Recommended workflow: build and position the scene in Phong first, then switch to Shadow Map to tune shadow-casting lights and verify the final look.

Main Menu Bar

File

This menu is the starting point for scene-level actions. In the current code it contains options like Nuevo and Abrir.

Windows

This is the central menu to open or close editor panels. From here you can toggle:

  • Performance
  • Details
  • Outliner
  • Fly cam config
  • Spawn entities
  • Explorer
  • Editor Input Bindings
  • Gizmo Controls

It also includes Save Scene, which serializes the current scene through the XML manager.

Panel-by-Panel Guide

1. Entity Hierarchy

This is your outliner. It shows scene entities using their transform hierarchy, so it is the main place to inspect parent-child relationships.

You use it to:

  • select an entity
  • inspect the scene tree
  • drag entities onto other entities to re-parent them
  • drop entities into the empty root area to make them root objects again

2. Details

This is the inspector for the currently selected entity. It is the most important editing panel in the whole editor.

Depending on the selected entity, it lets you edit:

  • Transform: position, rotation and scale
  • Render: visibility, color, texture, mesh path, drag & drop mesh replacement
  • Light: light type, colors, direction, attenuation, spot cone, shadows
  • Camera: camera properties
  • Script: script path and scripting state
In practice, the workflow is: select an entity in Hierarchy, then do all entity-level editing in Details.

3. Spawn

This panel is used to create new scene objects quickly from the editor.

You use it to spawn:

  • basic shapes
  • lights

Spawned entities are positioned using the current camera position, so it is useful for fast scene blocking.

4. Explorer

This is the file explorer panel. It recursively displays files and folders and allows drag & drop of resources into the editor.

In the current implementation it is especially useful for:

  • dragging .obj meshes into the Render section
  • dragging .png textures into render slots

5. Player

This panel controls runtime state inside the editor.

  • Play: starts runtime mode and serializes the current scene first
  • Pause: pauses and unpauses play mode
  • Stop: restores the serialized editor scene

This is very useful for testing scripts or runtime logic without permanently modifying the edited scene.

6. Performance

This panel displays FPS, frame time and a small historical graph.

Use it while testing Phong or Shadow Map scenes to check whether extra lights, shadows or heavy meshes are hurting performance.

7. Fly Cam Configuration

This panel is used to tune the fly camera behavior inside the editor.

It is especially useful when navigating large scenes, checking hierarchy placement or placing lights for shadow mapping.

8. Editor Input Bindings

This panel shows the editor key bindings currently registered in the input system.

It helps you verify what actions are mapped to what keys, especially for editor movement and gizmo controls.

9. Gizmo Mode

This panel controls transform gizmo behavior.

  • choose transform space: World or Local
  • adjust translate sensitivity
  • adjust rotate sensitivity
  • adjust scale sensitivity

This panel works together with the selected entity and the transform gizmo overlay.

10. Title Scene

This is the startup screen for creating or loading scenes.

It acts like the editor entry screen, with options such as:

  • NEW SCENE
  • LOAD SCENE

Simulated Workflow: Creating a Basic Lit Scene

This is a realistic editor workflow based on the current panel design. 

1. Open the editor and enter a new scene.
2. Open Windows > Outliner, Details, Spawn, Explorer, Performance and Player.
3. In Spawn, create a basic shape to use as the first object.
4. Select the entity in Entity Hierarchy.
5. In Details > Transform, move and scale the object.
6. In Details > Render, assign a mesh or replace it by dragging an .obj from Explorer.
7. In Spawn, create a Directional Light.
8. Select the light in Hierarchy.
9. In Details > Light:
   - set the type to Directional
   - enable it
   - tune direction
   - enable Casts Shadows if you want Shadow Map rendering
10. Use Fly Cam Configuration and editor input to navigate the scene.
11. Check Performance while adding more objects or lights.
12. Press Play to test runtime behavior.
13. Press Stop to restore the original editor state.
14. Use Save Scene when the layout is ready.

Simulated Workflow: Testing Shadows

This is the recommended sequence for a shadow setup. 

1. Build the scene first in Phong mode.
2. Add a light entity from Spawn.
3. Select the light in Hierarchy.
4. In Details > Light:
   - choose Directional or Spot
   - enable Casts Shadows
   - adjust Shadow Near / Shadow Far
   - adjust direction or cone values
5. Position scene meshes using Transform and hierarchy.
6. Use Fly Camera to inspect the scene from multiple angles.
7. Watch Performance to see the cost of the shadowed setup.
8. Save the scene when the shadow setup is stable.
Phong is your working mode for general look-dev. Shadow Map is your validation mode for shadowed lights.

How the Panels Work Together

  • Hierarchy + Details: selection and editing loop
  • Explorer + Details: drag & drop assets onto render fields
  • Spawn + Hierarchy: create entities, then organize them
  • Player + Scripts: test runtime logic safely
  • Performance + Light/Render: optimize as you build the scene
  • Gizmo + Transform: precise manipulation of selected entities
