Editor Panels

This page explains the current editor workflow based on the latest ImGui manager implementation. MentatEngine is centered around ECS editing, scene authoring, runtime testing and asset assignment through the editor UI.

Render Workflow

The engine currently revolves around two main rendering paths:

  • Phong: the default lit workflow for scene building and material/light iteration.
  • Shadow Map: the path used to validate shadow-casting light setups.
Recommended flow: build and tune the scene in Phong first, then validate light direction, shadow range and overall readability with Shadow Map.

Main Menu Bar

Scene

Scene-level actions. The current implementation includes:

  • Save Scene
  • Exit to Menu

Use this menu when you want to persist the current editor state or return to the title scene flow.

Windows

This menu toggles the main editor panels:

  • Performance
  • Details
  • Outliner
  • Fly cam config
  • Explorer
  • Editor Input Bindings
  • Gizmo Controls

Entity

This menu is for quick entity creation at the current camera position.

You can spawn:

  • Cube
  • Sphere
  • Pyramid
  • Horse
  • Square
  • Triangle
  • Ambient light
  • Directional light
  • Spot light
  • Point light

Panels

Title Scene

This is the scene-entry screen. It provides two main paths:

  • NEW SCENE: starts from the default scene template
  • LOAD SCENE: opens an existing scene from deps/resources/scenes

This makes the editor feel like a full scene launcher rather than just a raw debug panel.

Entity Hierarchy

This is the outliner of the scene. It shows entities as a hierarchy and supports parent-child organization.

You use it to:

  • select entities
  • rename visible scene structure mentally through tags
  • drag entities onto others to create parent-child relationships
  • drop entities into the root area to unparent them
  • toggle render visibility with the eye icon

Details

This is the main inspector for the selected entity.

It can expose and edit these sections:

  • Transform
  • Render
  • Light
  • Camera
  • Script

It also lets you:

  • delete the selected entity
  • remove specific components
  • add new components from the Add Component flow

Render section inside Details

This section is especially important for mesh-driven entities.

It supports:

  • visibility toggling
  • color activation
  • texture drop targets
  • mesh drop targets

This means the Explorer and Details panels work together directly for asset assignment.

Light section inside Details

This is where you configure all light behavior.

It includes controls for:

  • enabled state
  • shadow casting
  • direction normalization
  • quick directional presets like point down / point forward
  • attenuation presets
  • spotlight cone presets
This is the section most directly tied to the Phong vs Shadow Map workflow.

Script section inside Details

This section is where script-driven behavior is assigned to entities.

It includes a script drop target, so it fits naturally with the engine scripting workflow in deps/resources/scripts.

Player

This panel controls runtime testing inside the editor.

  • Play: serializes the current scene, starts play mode and switches editor styling
  • Pause: toggles runtime pause
  • Stop: restores the serialized scene state

This is especially useful for testing gameplay scenes like Pong without permanently modifying the authoring state.

Performance

Shows FPS, frame timing and a historical FPS plot.

Use it when comparing a simple Phong scene against a more expensive shadowed scene.

Fly cam configuration

Used to tune editor navigation. It includes reset controls and camera activation.

It is especially useful for large scenes or when inspecting lights and gameplay spaces.

Editor Input Bindings

Displays editor-side input bindings so you can verify navigation and tool controls.

Gizmo mode

Controls transform gizmo behavior and editing sensitivities.

This is the panel to use when fine-tuning transform editing speed while placing entities.

Explorer

Recursively browses assets and files. It is designed to work with drag & drop into other panels.

In practice, this is the bridge between your resources and entity setup.

Practical Workflow: Authoring a Scene

1. Open the Title Scene.
2. Create a new scene or load an existing one.
3. Open Entity Hierarchy, Details, Explorer, Performance and Player.
4. Spawn a mesh entity from the Entity menu.
5. Select it in Entity Hierarchy.
6. Edit position, rotation and scale in Details > Transform.
7. Assign or replace mesh/texture using Explorer + drag & drop.
8. Add a light from the Entity menu.
9. Tune the light in Details > Light.
10. Use Fly Cam to inspect the result.
11. Press Play to test runtime behavior.
12. Press Stop to recover the serialized editor scene.
13. Save Scene when the setup is ready.

Practical Workflow: Testing PongGame

1. Open the Title Scene.
2. Choose LOAD SCENE.
3. Load PongGame.
4. Open Entity Hierarchy, Details, Player and Performance.
5. Inspect script-driven entities like the ball and paddles.
6. Press Play to run the game logic.
7. Use Pause if you want to inspect the state mid-game.
8. Press Stop to restore the scene to its original editor state.
PongGame is now a very good showcase scene because it combines ECS entities, transforms, scripts, runtime state and editor-side testing.
